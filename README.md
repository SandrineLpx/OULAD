# OULAD Early Withdrawal Prediction

## Business Problem
Open University modules see a 31% withdrawal rate. Identifying at-risk students
early enough for intervention — before disengagement is entrenched — requires a
predictive model that works from the first weeks of study, not after a student
has already withdrawn.

## Dataset
**Open University Learning Analytics Dataset (OULAD)** — 32,593 student-module
enrolments across 7 modules and 4 presentations (2013B, 2013J, 2014B, 2014J) spanning 2 calendar years (2013-2014).
Sources: student demographics, VLE click logs, and assessment records.

## Approach: 21-Day Early Warning Cutoff
All features are windowed to the **first 21 days** of each module presentation.
This mirrors a realistic intervention timeline — a support adviser can act on
week-3 signals before withdrawal becomes likely. Features derived from any
later date are excluded by design.

## Leakage Prevention
| Source | Decision |
|---|---|
| Whole-course click aggregates (`total_clicks`, 18 per-activity columns) | Dropped — encode the full module, not the 21d window |
| `has_unregistered_flag` | Dropped — direct withdrawal proxy |
| Ratio features using whole-course denominator (`first_21d_click_share`) | Dropped |
| Redundant columns (r >= 0.99 pairs, near-zero variance) | Dropped — 15 columns removed |
| `code_presentation` (encodes year) | Kept for joins/analysis; excluded from model features |
| Semester component of presentation | Extracted as `is_feb_start` (B = Feb, J = Oct) — stable across years |
| Student-level train/test contamination (3,538 students in >1 enrolment) | Fixed via `GroupShuffleSplit` + `StratifiedGroupKFold` |
These controls prevent both target leakage and student-level contamination.

## Methods
| Component | Detail |
|---|---|
| **Models** | Logistic Regression (+ L1 Lasso, L2 Ridge), Decision Tree, Random Forest, Gradient Boosting, LightGBM, Neural Network MLP (Keras), Baseline dummy |
| **Validation** | Stratified 5-fold with group constraint (no student leaks across folds) |
| **Selection metric** | PR AUC — reflects performance on the minority withdrawal class |
| **Threshold tuning** | Optimal F1 threshold + precision >= 70% high-confidence threshold |
| **Calibration** | Isotonic post-hoc calibration; Brier score reported before/after |
| **Temporal holdout** | Train 2013, test 2014 — confirms generalisation to new cohorts |
| **SHAP** | Global bar + beeswarm, dependence plots, local waterfall (TP/TN/FN), group contribution |
| **Fairness** | Recall and FPR reported by module, IMD band, age band, and prior education |

## How to Run Locally
```bash
# 1. Install dependencies
pip install -r requirements.txt

# 2. Run the full offline pipeline (train, evaluate, SHAP)
python -m src.run_pipeline

# 3. Train the neural network separately (requires keras + torch)
python -m src.neural_network

# 4. Launch the dashboard (read-only — no retraining in the app)
streamlit run streamlit_app.py
```
